Chapter 10 - Debugging

Exception is an error that produces a return value that can be used by the program to deal with the error

Exceptions produce stack traces. Sequence of functions or method calls that lead to the point where the error occurred.

Warnings can occur if there's an error that's not enough to cause the program to crash.

Goal as a programmer is to write code that gives "loud" errors so they can be fixed during development, and when in production doesn't fail the user's experience

Add 'use strict'; to the first line of JavaScript file to make errors easier to spot, make things more clear, and keep your code not "sloppy"

Use strict can also be used inside a function and will only work inside that function

Test quality of JavaScript code by using JS Lint, JS Hint, or ES Lint. They will enforce a style guide, and highlight sloppy code as you type, or give you feedback depending on how you use it.

Use feature detection before using a browser feature to make sure it will work

Modernizr is a library that offers an easy way to implement feature detection and Can I Use? Is another useful resource

Alert() is a useful way to quickly debug, but there are much better tools now.

Console.log() is another good one. Does not stop execution of the program

Console.trace() will log an interactive stack trace in the console

You can set breakpoints in your browser

Debugger keyword will automatically create a breakpoint in your code that will pause the program at that point

This seems dumb because you can do it without a keyword and then will have to remove them when you're done. Just use the browser.

Error objects are created in the host environment when an exception occurs.

Const error = new Error(); // Can also have text as a parameter

EvalError - only around for backwards compatibility

RangeError - thrown for a number outside allowable range of values

ReferenceError - thrown when there's a reference to an item that doesn't exist

SyntaxError - error in the syntax

TypeError - thrown when there's an error in the type of value used i.e. string instead of number

URIError - thrown when there's a problem encoding or decoding the URI

// What's the URI?

InternalError - non-standard error that is thrown when an error occurs in the JavaScript engine. Can happen when using too much recursion

Error objects have properties that are used inconsistently across browsers, but safe ones are:

Name

Message

Stack - non-standard property, not safe to use in production sites

Throw statement can be used to throw exceptions.

i.e. throw new Error('Something is wrong');

Example:

    
        function squareRoot(number) {
            'use strict';
            if (number < 0) {
            throw new RangeError('You can't find the square root of negative numbers')
            }
            return Math.sqrt(number);
            };

Use exception handling to catch errors so they don't crash your program

Wrap code that may result in an exception in a 'try' block. If an exception occurs it will pass error object thrown into a 'catch' block


    function imaginarySquareRoot(number) {
        'use strict';
        try {
        return String(squareRoot(number));
        } catch(error) {
        return squareRoot(-number)+'i';
        }
        }

Test-driven development is the process of writing tests before any code.

  • Write tests
  • Write code to pass the tests
  • Refactor the code
  • Test refactored code
  • Write more tests for new features

Jest is a TDD framework that makes it easy to create and run tests.
